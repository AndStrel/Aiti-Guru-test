# Aiti Guru Test

Административная панель товаров на `React + TypeScript + Vite`.

Сейчас в проекте реализованы:
- Шаг 1: базовая структура, роутинг, подготовка к GitHub Pages.
- Шаг 2: авторизация через DummyJSON (`Zustand + Ant Design`).
- Шаг 3: таблица товаров через `React Query + Ant Design Table`.
- Дополнительно: регистрация пользователя через DummyJSON (`/users/add`).
- Дополнительно: локальное хранение `логин/пароль` зарегистрированного пользователя для повторного входа.

## 4. Как работает авторизация

### Маршруты

- `/#/login` — публичная страница входа.
- `/#/register` — публичная страница регистрации.
- `/#/products` — защищенная страница с таблицей товаров.
- `/#/` — автоматический редирект:
  - если пользователь авторизован -> `/#/products`
  - если нет -> `/#/login`

### Логика "Запомнить меня"

- Если чекбокс включен: сессия сохраняется в `localStorage`.
- Если чекбокс выключен: сессия сохраняется в `sessionStorage`.
- При сохранении очищается альтернативное хранилище, чтобы не было конфликтов.

### Локальные данные регистрации

- При регистрации `логин/пароль` записываются в `localStorage` и в Zustand-store.
- После выхода из аккаунта повторный вход проверяет данные с формой регистрации:
  - если логин совпал с зарегистрированным, пароль сверяется локально;
  - если пароль неверный, вход отклоняется без запроса к API.
- При перезагрузке страницы store восстанавливает зарегистрированные данные из `localStorage`.

### Обработка ошибок входа

- Ошибка показывается под формой.
- Дополнительно показывается `toast` через `message.error`.

### Как работает регистрация

- На странице логина есть ссылка `Создать`.
- Регистрация отправляется в `https://dummyjson.com/users/add`.
- После успешной регистрации пользователь сразу авторизуется и попадает на `/products`.

## 5. Подготовка к GitHub Pages

В `vite.config.ts` используется:

```ts
base: '/<repo-name>/'
```

Перед деплоем замените `<repo-name>` на имя вашего репозитория.

Для этого проекта обычно будет:

```ts
base: '/Aiti-Guru-test/'
```

Роутинг сделан через `HashRouter`, поэтому обновление страницы на GitHub Pages не приводит к 404.
