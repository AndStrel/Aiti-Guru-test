# Aiti Guru Test

Административная панель товаров на `React + TypeScript + Vite`.

Сейчас в проекте реализованы:
- Шаг 1: базовая структура, роутинг, подготовка к GitHub Pages.
- Шаг 2: авторизация через DummyJSON (`Zustand + Ant Design`).
- Шаг 3: таблица товаров через `React Query + Ant Design Table`.
- Шаг 4: поиск и сортировка с хранением состояния в URL.
- Шаг 5: добавление товара через модальное окно и форму Ant Design.
- Дополнительно: регистрация пользователя через DummyJSON (`/users/add`).
- Дополнительно: локальное хранение `логин/пароль` зарегистрированного пользователя для повторного входа.

## 4. Как работает авторизация

### Маршруты

- `/#/login` — публичная страница входа.
- `/#/register` — публичная страница регистрации.
- `/#/products` — защищенная страница с таблицей товаров.
- `/#/` — автоматический редирект:
  - если пользователь авторизован -> `/#/products`
  - если нет -> `/#/login`

### Логика "Запомнить меня"

- Если чекбокс включен: сессия сохраняется в `localStorage`.
- Если чекбокс выключен: сессия сохраняется в `sessionStorage`.
- При сохранении очищается альтернативное хранилище, чтобы не было конфликтов.

### Локальные данные регистрации

- При регистрации `логин/пароль` записываются в `localStorage` и в Zustand-store.
- После выхода из аккаунта повторный вход проверяет данные с формой регистрации:
  - если логин совпал с зарегистрированным, пароль сверяется локально;
  - если пароль неверный, вход отклоняется без запроса к API.
- При перезагрузке страницы store восстанавливает зарегистрированные данные из `localStorage`.

### Обработка ошибок входа

- Ошибка показывается под формой.
- Дополнительно показывается `toast` через `message.error`.

### Как работает регистрация

- На странице логина есть ссылка `Создать`.
- Регистрация отправляется в `https://dummyjson.com/users/add`.
- После успешной регистрации пользователь сразу авторизуется и попадает на `/products`.

### Поиск и сортировка товаров (Шаг 4)

- Поиск выполняется через `https://dummyjson.com/products/search?q=...`.
- Если поле поиска пустое, загружается общий список из `https://dummyjson.com/products`.
- Сортировка доступна по колонкам `Цена` и `Оценка`.
- Состояние поиска и сортировки хранится в URL-параметрах:
  - `q` — текст поиска;
  - `sortBy` — поле сортировки (`price` или `rating`);
  - `order` — направление (`asc` или `desc`).

### Добавление товара (Шаг 5)

- Над таблицей есть кнопка `Добавить`.
- По кнопке открывается `Modal` с формой.
- Обязательные поля: `Наименование`, `Цена`, `Вендор`, `Артикул`.
- Реальный POST-запрос не выполняется (по ТЗ).
- После добавления товар попадает в локальный стейт текущей сессии и сразу отображается в таблице.
- После submit:
  - модалка закрывается;
  - форма очищается;
  - показывается `message.success('Товар успешно добавлен')`.

Сейчас в `vite.config.ts` уже установлен:

## 7. Что уже проверено

- `npm run lint` проходит без ошибок.
- `npm run build` проходит успешно.
- Неавторизованный пользователь не может попасть на `/products`.
- Успешный логин переводит на `/products`.
- После регистрации пользователь сразу попадает на `/products`.
- После логаута зарегистрированный пользователь может снова войти по локально сохраненным данным.
- После перезагрузки страницы локальные регистрационные данные остаются доступны.
- Таблица товаров загружается с `https://dummyjson.com/products`.
- В таблице есть колонки: `Наименование`, `Вендор`, `Артикул`, `Оценка`, `Цена`, `Количество`.
- В колонке "Оценка" значения ниже 3 подсвечиваются красным.
- Поиск и сортировка корректно отражаются в URL и сохраняются при обновлении страницы.
- Модальное окно добавления товара работает по UX-сценарию из ТЗ (закрытие/очистка/success toast).
- Добавленный вручную товар сразу участвует в таблице, поиске и сортировке в рамках текущей сессии.
